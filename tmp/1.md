# lazycc 成熟化目录拆分与 CLI Tool Capability Trait 方案

## Summary

将项目按能力边界拆分，同时把当前散落在 `config.rs` 和 `tui/state.rs` 中的 Codex/Claude CLI
差异抽象出来。核心目标不是只抽 MCP，而是先定义“外部 coding-agent CLI 工具”的通用 trait，再把
MCP list/remove、shell wrapper、环境变量映射等能力作为 capability 挂到具体工具实现上。

这样后续新增其他 CLI 工具，或者给 Codex/Claude 扩展新的能力时，不需要继续把工具差异写进 TUI
状态逻辑、shell 输出逻辑或 `Target` 的 match 分支里。

## Key Changes

- `main.rs` 收敛为薄入口，只调用 `cli::run()` 并处理错误退出。
- `cli` 负责 Clap 命令定义、命令分发和 inquire 交互输入。
- `domain` 负责 `Target`、`Shell`、`Profile`、`CurrentProfile`、`Config` 以及 profile
  增删改用等业务规则。
- `storage` 负责 config 路径、TOML 读写和文件权限。
- `shell` 负责 `lazycc init zsh` 和 `lazycc` shell wrapper 输出编排。
- `output` 负责 `render_table`、`mask_api_key` 和 comfy-table 展示逻辑。
- `tui` 继续负责终端 UI，但从 `state.rs` 中移出 Codex/Claude CLI 命令执行、解析和工具差异。
- 新增 `tools` 能力模块，承载 CLI 工具 trait、能力 trait、命令 runner、registry，以及 Codex/Claude
  实现。

## CLI Tool Trait Design

`Target` 保留为配置文件和 CLI 参数中的稳定枚举，但不再承载具体工具能力。工具能力从
`Target::{env_vars, model_env_var, mcp_command}` 迁移到 `CodingAgentTool` 及其 capability 上。

```rust
pub(crate) trait CodingAgentTool {
    fn target(&self) -> Target;
    fn id(&self) -> &'static str;
    fn display_name(&self) -> &'static str;
    fn command(&self) -> &'static str;

    fn env(&self) -> &dyn EnvCapability;
    fn shell(&self) -> Option<&dyn ShellCapability>;
    fn mcp(&self) -> Option<&dyn McpCapability>;
}
```

- `id()` 用于内部注册和错误信息，例如 `codex`、`claude`。
- `display_name()` 用于表格和 TUI 展示，例如 `Codex`、`Claude Code`。
- `command()` 是真实外部命令，例如 `codex`、`claude`。
- capability 使用 `Option` 表示工具不一定支持所有能力，便于后续新增只支持部分功能的 CLI。

## Capability Trait Design

### EnvCapability

```rust
pub(crate) trait EnvCapability {
    fn base_url_env(&self) -> &'static str;
    fn api_key_env(&self) -> &'static str;
    fn model_env(&self) -> Option<&'static str>;
    fn all_envs(&self) -> &'static [&'static str];
}
```

- 替代当前 `Target::env_vars()`、`Target::model_env_var()` 和 `Target::all_env_vars()`。
- shell 初始化时从工具 registry 汇总所有 env，避免新增工具时遗漏 unset。

### ShellCapability

```rust
pub(crate) trait ShellCapability {
    fn zsh_wrapper(&self, profile: &Profile) -> Option<String>;
}
```

- 替代当前只针对 Codex 的 `push_codex_wrapper()` 逻辑。
- Codex 可以在非默认 profile 下生成 `codex()` wrapper。
- Claude 当前可以返回 `None`，未来如果需要 wrapper 不影响调用方结构。

### McpCapability

```rust
pub(crate) trait McpCapability {
    fn list_servers(&self) -> Result<Vec<McpServer>, ToolError>;
    fn remove_server(&self, name: &str) -> Result<(), ToolError>;
}
```

- `McpServer { target, name, details }` 从 `tui/state.rs` 移到 `tools::capabilities::mcp`。
- `ToolError` 封装命令启动失败、非零退出和解析错误，错误消息保留工具 id/command 便于 TUI 展示。
- Codex MCP 执行 `codex mcp list` / `codex mcp remove <name>`。
- Claude MCP 执行 `claude mcp list` / `claude mcp remove <name>`。
- 当前只实现 list/remove，但 trait 位置允许后续扩展 add/get/update 等 MCP 操作。

## Registry And Runner

```rust
pub(crate) fn tool_for(target: Target) -> &'static dyn CodingAgentTool;
pub(crate) fn all_tools() -> &'static [&'static dyn CodingAgentTool];
```

- TUI 通过 `all_tools()` 刷新所有支持 MCP 的工具，不再遍历 `Target::all()` 后手写命令。
- shell 初始化通过 `all_tools()` 汇总 env 和 wrapper。
- table 渲染通过 `tool_for(profile.target).display_name()` 展示目标名称。
- 新增 command runner 抽象，测试命令构造和输出解析时不真实依赖本机 `codex` / `claude`。

## Migration Order

1. 新增 `tools` 模块和 registry：先迁移 `Target::display_name()`、CLI command 名称、env 映射，保持行为不变。
2. 抽出 command runner 和 `McpCapability`：移动 `McpServer`、list/remove、解析逻辑，让 TUI 只依赖
   `tool.mcp()`。
3. 抽出 `ShellCapability`：把 Codex wrapper 生成从 `config.rs` 迁到 Codex tool 实现。
4. 拆 `config.rs`：迁移到 `domain`、`storage`、`shell`、`output`，保持配置格式和 shell 输出不变。
5. 拆 `main.rs`：迁移 Clap 类型、命令分发、交互 prompt 到 `cli`。
6. 拆 `tui/state.rs`：将 `TuiApp`、`ProfileForm`、键鼠输入处理分离到 `app/form/input`。
7. 收紧可见性：优先使用 `pub(crate)`，只暴露跨模块真正需要的类型和函数。

## Test Plan

- 每一步迁移后运行 `cargo test`，确保行为不变。
- 新增工具抽象测试：
  - `tool_for(Target::Codex)` 返回 Codex tool，command 为 `codex`。
  - `tool_for(Target::Claude)` 返回 Claude tool，command 为 `claude`。
  - registry 中每个 tool 的 `target()` 唯一，且覆盖 `Target::all()`。
  - env capability 能汇总 Codex/Claude 的所有 env，并用于 shell unset。
- 新增 MCP 测试：
  - Codex list 解析跳过空行、warning、表头，取第一列为 name。
  - Claude list 解析用 `": "` 前内容作为 name。
  - list/remove 命令构造通过 command runner 抽象测试，不真实依赖本机 `codex` / `claude`。
- 保留并迁移现有测试：
  - profile 重名拒绝、默认 profile 不可删除、target 解析。
  - shell init 输出、Codex wrapper、表格脱敏。
  - TUI profile 切换、添加、编辑、删除。
  - MCP pane 删除确认、`selected_mcp` / `mcp_index`、删除后刷新。
- 完成后运行：
  - `cargo fmt --check`
  - `cargo clippy --all-targets --all-features`
  - `cargo build --release --locked`

## Assumptions

- 本次是结构成熟化和能力抽象，不改变 CLI 命令、配置文件格式、TUI 行为或 README 描述的用户可见能力。
- `Target` 继续作为配置枚举存在，工具 trait 只负责把 `Target` 映射到可执行 CLI 和能力集合。
- capability 按“当前能力 + Future Ops”设计：当前落地 env、shell、MCP，后续可以扩展 auth、doctor、
  version、model discovery 等能力。
- 暂不引入 `lib.rs` 公共库 API；如未来需要集成测试或第三方复用，再提升核心模块为 library。
- 暂不引入复杂 service/repository trait，除 CLI tool/capability 外优先用清晰模块边界解决当前问题。
